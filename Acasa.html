    Quicksort

    Despre

    Este un algoritm de sortare care, pentru un sir de n elemente, are un timp de executie in cazul cel mai defavorabil O(n2), a fost Dezvoltat de informaticianul britanic Tony Hoare in 1959 si publicat in 1961 si cand este implementat bine, poate fi ceva mai rapid decat Merge Sort si de aproximativ doua sau trei ori mai rapid decat Heap Sort. La fel ca Merge Sort, Quicksort este un algoritm de sortare bazat pe abordarea Divide and Conquer.

    Functioneaza prin selectarea unui element „pivot” din matrice si partitionarea celorlalte elemente in doua sub-matrice, in functie de faptul ca acestea sunt mai mici sau mai mari decat pivotul. Din acest motiv, uneori se numeste sortare cu schimb de partitii. Sub-matricele sunt apoi sortate recursiv. Acest lucru se poate face in loc, necesitand cantitati suplimentare mici de memorie pentru a efectua sortarea.

    Quicksort este o sortare de comparatie, ceea ce inseamna ca poate sorta articole de orice tip pentru care este definita o relatie „mai putin decat” (formal, o ordine totala). Implementarile eficiente ale Quicksort nu sunt o sortare stabila, ceea ce inseamna ca ordinea relativa a elementelor de sortare egala nu este pastrata. Analiza matematica a sortarii rapide arata ca, in medie, algoritmul ia O ( n log ⁡ n ) O(n\log {n}) comparatii pentru a sorta n articole. In cel mai rau caz, face O ( n 2 ) O(n^{2}) comparatii.

    Avantaje:

    1. Sorteaza pe loc, deoarece foloseste doar o mica stiva auxiliara.
    2. Este nevoie doar de n (log n) timp pentru a sorta n articole.
    3. Are o bucla interioara extrem de scurta.
    4. Acest algoritm a fost supus unei analize matematice amanuntite, se poate face o afirmatie foarte precisa despre problemele de performanta.

    Dezavantaje:

    1. Este recursiv. Mai ales, daca recursiunea nu este disponibila, implementarea este extrem de complicata.
    2. Este nevoie de timp patratic (adica, n2) in cel mai rau caz.
    3. Este fragil, adica o simpla greseala in implementare poate trece neobservata si poate duce la functionarea proasta.

    Algoritm quicksort

    Partitionarea sirului

      quicksort(a, inf, sup)
      1. daca inf < sup atunci
      2. q <- partitie(a, inf, sup)
      3. quicksort(a, inf, q)
      4. quicksort(a, q + 1, sup)
      sfarsit daca

      partitie(a, inf, sup)
      1. x <- a[inf]
      2. i <- inf – 1
      3. j <- sup + 1
      4. cât timp adevărat execută
      5. executa
      6. j <- j – 1
      7. cat timp a[j] > x
      8. executa
      9. i <- i + 1
      10. cat timp a[i] < x
      11. daca i < j atunci
      12. interschimba a[i] <-> a[j]
      13. altfel
      14. returneaza j
      sfarsit daca sfarsit cat timp
